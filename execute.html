03
ANALYTICAL DRAFTING
Draft the full BriefCycle JSON using Claude's native analytical capability
This is the core analytical phase. Read the tagged items, read all 11 prompt templates in pipeline/draft/prompts/, read the previous brief from briefs/ for delta context, then draft the complete brief. Follow every rule in CLAUDE.md — especially:

VOICE RULES (NON-NEGOTIABLE)

RULECORRECTVIOLATION
Assessment-first leads "Offensive preparations appear underway; three BTGs have repositioned." "Three BTGs were observed near Kherson."
No first-person plural "Reporting suggests…", "Evidence indicates…" "We assess…", "We judge…", "We evaluate…"
Because-clause for hedging "Escalation is likely, given simultaneous IRGC repositioning (Reuters, 15 Mar)." "Escalation is likely."
Sentence discipline Noun + verb in first 6 words. Target 12–20 words/sentence. "It is important to note that the situation is evolving."
Iranian state media "Iranian government asserts…" — verification: "claimed" Citing IRNA/PressTV as confirmed fact

CONFIDENCE LANGUAGE (ONLY THESE 6 — NO AD-HOC HEDGING)

ENUMRENDERED PHRASEPROBABILITY
almost-certainlyDirect declarative / "Evidence strongly indicates…"95–99%
highly-likely"Multiple sources confirm…" / "Balance of reporting points to…"75–95%
likely"Reporting suggests…" / "Available evidence points to…"55–75%
possibly"Reporting indicates, though corroboration is limited…"45–55%
unlikely"This remains unlikely, though it cannot be excluded…"20–45%
almost-certainly-not"Nothing in the reporting supports…"1–5%

DRAFTING ORDER (EXACT)

  1. Read all 11 prompt templates from pipeline/draft/prompts/
  2. Read previous brief from briefs/ (latest JSON) for delta context
  3. Read pipeline/.cache/tagged_YYYYMMDD.json — group items by tagged_domains
  4. Draft domains in order: d1 → d2 → d3 → d4 → d5 → d6
  5. Draft flash points (Tier 1 only or high-significance multi-corroborated)
  6. Draft executive summary (BLUF + key judgments + KPIs — synthesize across domains)
  7. Draft strategic header (headline judgment + trajectory + threat level)
  8. Draft warning indicators (8 pre-defined tripwires)
  9. Draft collection gaps (genuine intelligence limits — no padding)
  10. Assemble meta, caveats, footer

DOMAIN SECTION STRUCTURE (EACH d1–d6)

Each domain section must contain:
  • id: d1–d6
  • num: "01"–"06"
  • title: domain name in CAPS
  • assessmentQuestion: analytical question from the prompt template
  • confidence: one of the 6 enum values
  • keyJudgment: { id, domain, confidence, probabilityRange, language, text, basis, citations }
  • bodyParagraphs[]: each with subLabel, subLabelVariant (observed|assessment), text, timestamp, citations, confidenceLanguage
  • Optional: dissenterNote, analystNote, tables, timeline, actorMatrix
Every claim must trace to a triaged item. No fabrication. If a domain has zero items, write a thin-data placeholder and flag the collection gap.

FORBIDDEN PHRASES (INSTANT FAIL)

kinetic activity · threat actors · threat landscape · robust · leverage (verb) · we assess · we judge · we evaluate · our assessment · diplomatic efforts · international community · stakeholders · going forward · ongoing situation · fluid situation · escalatory dynamics · economic headwinds · market volatility · risk environment · uncertain times · significant development · notable development · rapidly evolving · dynamic situation · heightened tensions · broader conflict · it should be noted · importantly · significantly · notably · at this juncture · complex situation · holistic approach · paradigm shift · game changer · due to the fact that · duties and responsibilities · importance and significance

MULTI-PASS AUTOMATED EXECUTION (RECOMMENDED)

For the best product, run 2–3 passes automatically. Each pass uses a separate Claude Code session via the CLI, so the full context window is available for each pass. No manual intervention required.
A
THREE-PASS ARCHITECTURE
Draft → Review + Fix → Polish + Publish
PASSPURPOSEFOCUS
1 Full Pipeline Ingest → Triage → Draft → Publish. Gets the brief written with all data.
2 Quality Review + Redraft Reads the brief as an editor. Checks 14 quality criteria:
• Forbidden phrase scan • Assessment-first leads • Because-clauses on hedging
• Confidence enum compliance • Sentence discipline • Structural completeness
• Citation integrity • Iranian source treatment • Cross-domain coherence
Fixes every issue directly in the JSON.
3 Final Polish + Commit Sentence-level copyedit: cut stretchers, fix nominalizations, enforce temporal precision, verify source attribution, check cross-domain coherence. Then export, build, commit, push.

RUN IT

# 2 passes (faster — skip the polish pass)
bash scripts/auto_cycle.sh 2

# 3 passes for a specific date
bash scripts/auto_cycle.sh 3 2026-03-15

HOW IT WORKS

The script spawns separate claude -p sessions for each pass. Each session gets the full context window — no degradation from prior passes.
  • Pass 1 runs /workflow end-to-end (ingest → triage → draft → publish)
  • Pass 2 reads the output brief + CLAUDE.md + all prompt templates, audits 14 quality criteria, fixes issues in-place
  • Pass 3 does sentence-level copyediting, re-exports HTML/DOCX, commits and pushes to main
Logs are saved to scratch/auto_cycle_logs/ for audit. The script never stops on errors — degraded output always beats no output.
